还有就是
鼠标-主设备-从设备-电脑 这个链路的延迟会怎样
  蓝牙鼠标→nRF：约 5～10ms（蓝牙 BLE 1M，100Hz～1kHz）
  nRF 内部处理 + 坐标累加：<1ms
  nRF→USB HID→电脑：约 1～8ms（取决于 USB 轮询率：125Hz≈8ms，1000Hz≈1ms）
  系统 + 显示器：16.7ms（60Hz）/ 6.9ms（144Hz）
延迟可以再降低些吗
  总延迟
  6ms ~ 10ms

主板到从板的数据传输 你是把数据再次封装了下吗？因为后期会添加键盘功能，所以数据不能直接转发的形式， 否则会和键盘的数据冲突 从设备分不出是鼠标数据还是键盘数据

还有点细节， 主板并不是把所有的鼠标和键盘的事件都发给从板。 主板根据鼠标当前的移动位置，决定了鼠标键盘的事件是主板自己处理还是发给从板处理。 这样的情况， 主板内部应该建立鼠标的位置系统。假设主板和从板连接的都是1920x1080的分辨率的电脑（这个分辨率可以在主板通过webhid配置）， 当主板没有连接上从板， 模拟的分辨率应该是1920x1080，当连接上从板后， 模拟的分辨率应该是3840x1080，也就是两个分辨率水平排列， 当然分辨率是可配置的，主从组合的模拟分辨率也应该是实时计算出来的。 主板通过计算鼠标当前的位置，把键鼠信息发给主板或从板。这样的话 鼠标的移动也应该是类似数控版的绝对位置，是吧
